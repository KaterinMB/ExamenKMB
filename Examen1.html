Explicación breve sobre formularios

Form Anida todos los elementos, posee el action ymethod .
Imput Posee el type que es un campo obligatorio, el id obligatorio si en label hay un atributo for, name y value.
Select Permite crear una lista desplegable de opciones hace uso de id, name.
Option Encapsula cada opción de la lista. Utiliza el Value
Textarea Es un campo de texto, normalmente utilizado para biografias o descripciones.
Label Especifica la etiqueta del campo del formulario.
Button
 Hace uso del type: puede contener submit, reset, button.
(Opcionalmente) required: indica que el campo debe ser llenado.
readonly: que el campo es solo para lectura.
placeholder: aparece un texto de ayuda para rellenar el campo.
value:para introducir un valor por defecto.

Para representar Entidades

Si quisieramos hablar de la etiqueta hr, mostrandola con los signos de mayor y menor a como se ve a continuación (La etiqueta <hr> se utiliza para...)
es necesario modificar el símbolo "menor que" (Lower Than) por la entidad HTML lt y "mayor que" (Greater Than) por gt, así el navegador podrá representarlo sin ningún problema.

Anidación

En la descripción de todo elemento HTML está el content model que especifica qué tipos de etiqutas se pueden anidar.
Ejemplos: metadata content, flow content, sectioning content, heading content, phrasing content.

Validación

  1. Validate URI: que te permite introducir la URL de una página para comprobar su código.
  2. Validate by File Upload: que nos permite subir un fichero HTML.
  3. Validate by Direct Input: que nos permite introducir el código dentro de unelemento.
